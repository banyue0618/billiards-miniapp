#### 域名/路径解析租户（最推荐生产）

- 做法：给每个租户配二级域名或绑定自定义域，如 acme.example.com → tenantId=acme；或使用路径前缀 /t/{tenant}/login。

- 优点：用户只输账号密码；安全、可伸缩；天然品牌隔离。

- 落地：网关/Nginx 解析 Host/Path → 写入请求头 X-Tenant-Id → 后端拦截器设置租户上下文并持久化会话。

- 结合现有的租户上下文与拦截器体系实现很顺滑（登录成功后 TenantHelper.setDynamic(tenantId, true) 会话持久化）

#### 两步登录（先识别租户，再校验口令）（当前实现方式）

- 做法：第一步输入“公司/租户标识”（租户编码、公司名）；系统确定租户后，第二步才输入账号密码。

- 优点：无域名治理成本；体验接近单租户。

- 注意：不要暴露全量租户列表，改为“远程模糊搜索”或“带参登录链接”。

#### 平台级账户 + 选择租户

- 登录接口只校验平台用户；登录成功跳转“选择租户页”，列出“我加入的租户”；选择后调用 /auth/switchTenant，由后端执行 TenantHelper.setDynamic(tenantId, true)，再进入后台首页。

- 适合“平台级 bls_user 账号不绑定商户”的架构设定。

#### **用户名全局唯一（带上租户标识）**

- 用户名在全局唯一，通常通过拼接规则：
  - `username@tenantA`
  - `username@company.com`
- 登录时只需输入这一串，系统就能拆出用户名和租户标识。

#### **邮箱 / 手机号全局唯一**

- 如果你要求注册时邮箱或手机号 **全局唯一**，那它就能作为定位用户的关键。
- 登录流程：
  1. 用手机号/邮箱直接查全局用户表。
  2. 反查所属租户，再进行密码验证。
      ✅ 优点：和常见平台（阿里云、AWS）类似，统一入口。
