##### 会员积分消费详情表(bls_member_points_consume_detail)

| 字段名      | 类型        | 说明                                         |
| ----------- | ----------- | -------------------------------------------- |
| id          | varchar(32) | 记录ID                                       |
| user_id     | varchar(32) | 用户ID                                       |
| record_id   | varchar(32) | 指向消费类 bls_member_points_record.id       |
| validity_id | varchar(32) | 指向被扣的批次 bls_member_points_validity.id |
| points      | int         | 本行扣减的积分（正数表示扣减量）             |

- 作用：把“一次消费记录”拆分到“多个有效期批次”的扣减明细，解决 FIFO 扣减与后续退款/回滚/审计问题。

- 背景：一次消费可能跨多个批次扣减。例如：

  - 批次A：剩余50，过期10-31

  - 批次B：剩余100，过期12-31

  - 本次消费120 → 实际扣减：A扣50、B扣70

  - 没有这张表时，bls_member_points_record 只有一条 “-120”，丢失“从哪些批次扣了多少”的信息：

    - 审计难以还原

    - 退款/撤销无法精确“加回原批次”（尤其要遵守未过期优先）

  - 有了明细表后，可记录两行：

    - record_id=消费记录，validity_id=A，points=50

    - record_id=消费记录，validity_id=B，points=70

    - 退款时按明细逐条回补对应批次（若批次已过期则按规则处理）。
