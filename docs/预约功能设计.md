## 小程序台球厅预约功能流程说明

### 🟢 1. 用户发起预约

**操作入口：** 用户选择门店，点击[预约]，进入小程序「预约页面」，选择桌台、预约时间段。预约页面显示当前空闲桌台的空闲预约时间段范围（后端需要查询当前桌台的预约记录）

**预约页面信息：**

- 门店显示可预约的桌台列表

- 每张桌台显示当前可预约时间段（根据营业时间、其他预约记录、预约最短时间三个参数生成24小时时间段，可预约显示绿色，不可预约显示红色）
- 用户选择桌台、时间进行预约
- 页面风格需要合理设计，复合系统整体的apple风格

**用户提交预约：**

 **输入项：**

- 台球桌 ID（table_id）
- 开始时间（start_time）
- 结束时间（end_time）

**后端校验逻辑：**

1. 检查该门店是否营业中；
2. 检查该台球桌在该时间段是否已有有效预约（通过sql检查是否存在时间段冲突）；
3. 检查用户当日预约次数、历史爽约次数、是否被封禁；
4. 校验时间范围是否合理（起止时间、营业时间、间隔是否满足最小单位，如30分钟）。

**若通过校验：**

- 生成预约编号（reservation_no）；
- 写入预约表（状态设为 `0=预约中`）；
- 若为定金模式，则返回支付信息（暂不支持）。

------

### 🟡 2. 预约支付阶段（暂不支持）

如果门店配置为“需定金预约”：

- 用户调起微信支付；
- 支付成功 → 更新 `pay_status=1`，`pay_time=now()`；
- 若超时未支付（如10分钟） → 预约失效，标记为 `3=已取消`。

------

### 🟠 3. 预约等待期

根据系统设置定时任务（如每5分钟，可配置）检测：

- 若当前时间超过 `start_time+10分钟（可配置）` 且未签到 → 状态改为 `4=已过期`；

------

### 🟢 4. 用户到店确认（开台）

**触发方式：**

- 用户进入小程序，首页显示：[我的预约]，操作直接点击开台。
- 用户进入门店选择扫码开台。

**后端逻辑：**

1. 扫码开台，检查当前用户是否存在预约记录，更新预约记录为已到店；
2. 预约开台，直接更新对应的预约记录为已到店（此时通过预约开台传递参数预约码）
3. 更新：
   - `status=1`（已到店）；
   - `check_in_time=now()`；

------

### 🔴 6. 用户取消预约

**前端条件：**

- 仅当 `status=0` 且未到达 `start_time` 才能取消。

**后端逻辑：**

1. 更新 `status=3`（已取消）；
2. 若有定金，根据规则执行退款；
3. 记录 `cancel_time`；
4. 更新台球桌状态为空闲。

------

## ✅ 状态流转总览（文字表）

| 阶段     | 状态值 | 说明             | 触发方式                         |
| -------- | ------ | ---------------- | -------------------------------- |
| 创建成功 | `0`    | 预约中           | 用户提交（预约开始）             |
| 到店签到 | `1`    | 已到店（使用中） | 扫码/签到（预约结束）            |
| 用户取消 | `3`    | 已取消           | 用户主动操作（预约结束）         |
| 超时未到 | `4`    | 已过期           | 用户定时任务自动处理（预约结束） |

## 预约配置参数

##### 🧩 一、时间相关配置

| 参数名                                    | 示例值 | 说明                                             | 级别   |
| ----------------------------------------- | ------ | ------------------------------------------------ | ------ |
| `reservation.min_duration_minutes`        | 30     | 每次最短预约时长（分钟）                         | 系统级 |
| `reservation.max_duration_minutes`        | 120    | 每次最长预约时长（分钟）                         | 系统级 |
| `reservation.advance_reserve_days`        | 3      | 用户可提前预约天数（自然日）                     | 系统级 |
| `reserve.expire_after_start_minutes`      | 5      | 预约开始后多少分钟未签到则自动过期并释放（分钟） | 系统级 |
| `reservation.remind_before_start_minutes` | 15     | 开始前多久发送提醒（分钟）                       | 系统级 |

------

##### 💰 二、支付/定金相关配置

| 参数名                                    | 示例值         | 说明                                            | 级别   |
| ----------------------------------------- | -------------- | ----------------------------------------------- | ------ |
| `reservation.require_deposit`             | true           | 是否启用定金模式                                | 门店级 |
| `reservation.deposit_amount`              | 10.00          | 定金金额（元）                                  | 门店级 |
| `reservation.deposit_refund_policy`       | `before_30min` | 退款策略（如：开始前30分钟可退、always、never） | 门店级 |
| `reservation.refund_grace_period_minutes` | 30             | 开始前多少分钟内取消不可退                      | 门店级 |

------

##### 🧍 三、用户限制类配置

| 参数名                                 | 示例值 | 说明                         | 级别   |
| -------------------------------------- | ------ | ---------------------------- | ------ |
| `reservation.max_per_day`              | 2      | 每个用户每日最多预约次数     | 系统级 |
| `reservation.max_pending_reservations` | 1      | 用户同时存在的进行中预约上限 | 系统级 |
| `reservation.ban_after_no_show_count`  | 3      | 连续爽约多少次后禁止预约     | 系统级 |
| `reservation.ban_days_after_no_show`   | 7      | 封禁时长（天）               | 系统级 |

------

##### 🧾 四、门店/资源层面配置

| 参数名                            | 示例值        | 说明                         | 级别   |
| --------------------------------- | ------------- | ---------------------------- | ------ |
| `reservation.enable_for_store`    | true          | 门店是否开启预约功能         | 门店级 |
| `reservation.max_tables_per_user` | 1             | 单次最多可预约的桌数         | 门店级 |
| `reservation.time_unit_minutes`   | 30            | 时间粒度（30分钟为一个单位） | 系统级 |
| `reservation.opening_hours`       | `10:00-23:00` | 预约可选时间段               | 门店级 |

------

##### 🔔 五、通知与提醒配置

| 参数名                                  | 示例值     | 说明                   | 级别   |
| --------------------------------------- | ---------- | ---------------------- | ------ |
| `reservation.enable_reminder`           | true       | 是否开启预约提醒       | 系统级 |
| `reservation.reminder_template_id`      | `WX123456` | 微信模板消息ID         | 系统级 |
| `reservation.reminder_before_minutes`   | 15         | 提前多久提醒           | 系统级 |
| `reservation.notify_on_cancel`          | true       | 是否推送取消通知       | 系统级 |
| `reservation.cancel_notify_template_id` | `WX123456` | 取消通知微信模板消息ID | 系统级 |

------

##### ⚙️ 六、后台/系统行为配置

| 参数名                                    | 示例值 | 说明                     | 级别   |
| ----------------------------------------- | ------ | ------------------------ | ------ |
| `reservation.auto_check_interval_minutes` | 5      | 后台定时任务执行间隔     | 系统级 |
| `reservation.allow_admin_override`        | true   | 管理员是否可手动调整预约 | 系统级 |
| `reservation.audit_log_enabled`           | true   | 是否记录预约操作日志     | 系统级 |
