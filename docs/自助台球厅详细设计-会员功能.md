### 10.7 核心接口设计

#### 10.7.1 会员信息接口
```java
/**
 * 获取会员信息
 * @param userId 用户ID
 * @return 会员信息VO
 */
MemberInfoVO getMemberInfo(String userId);
```
业务描述：
1. 获取用户当前等级配置信息
2. 查询用户基本信息（积分、累计消费等）
3. 获取适用的会员权益列表
4. 查询近期积分变动记录
5. 返回完整的会员信息VO，包括：
   - 基本信息（用户ID、等级名称等）
   - 当前积分和累计消费金额
   - 等级到期时间
   - 本月已使用和剩余免费时长
   - 适用的权益列表
   - 近期积分记录

#### 10.7.2 会员等级列表接口
```java
/**
 * 获取会员等级列表
 * @return 会员等级VO列表
 */
List<MemberLevelVO> getMemberLevels();
```
业务描述：
1. 查询所有有效的会员等级配置
2. 按所需消费金额升序排序
3. 转换为前端展示所需的VO对象，包括：
   - 等级基本信息（编码、名称、图标等）
   - 所需消费金额
   - 等级权益（折扣、积分倍率等）
   - 特权标识（VIP服务、预订特权等）
4. 用于展示会员等级体系和晋升条件

#### 10.7.3 会员权益列表接口
```java
/**
 * 获取会员权益列表
 * @param userId 用户ID
 * @return 会员权益VO列表
 */
List<MemberBenefitVO> getMemberBenefits(String userId);
```
业务描述：
1. 获取用户当前会员等级
2. 查询所有有效的会员权益（状态正常、在有效期内）
3. 过滤出适用于当前会员等级的权益
4. 转换为VO对象，包括：
   - 权益基本信息（名称、图标等）
   - 权益类型和权益值
   - 使用规则和说明
   - 有效期信息
   - 标签列表
5. 按权益排序号排序返回

#### 10.7.4 积分记录接口
```java
/**
 * 获取积分记录
 * @param userId 用户ID
 * @param pageNum 页码
 * @param pageSize 每页大小
 * @return 积分记录VO列表
 */
List<MemberPointsRecordVO> getPointsRecords(String userId, Integer pageNum, Integer pageSize);
```
业务描述：
1. 创建分页对象
2. 查询指定用户的有效积分记录
3. 按创建时间倒序排序
4. 转换为VO对象，包括：
   - 积分变动值（正数表示增加，负数表示减少）
   - 变动类型和场景
   - 关联的规则信息
   - 变动说明和时间
5. 支持分页查询，便于展示大量记录

#### 10.7.5 计算会员优惠金额接口
```java
/**
 * 计算会员优惠金额
 * @param userId 用户ID
 * @param originalAmount 原始金额
 * @return 优惠后金额
 */
Double calculateDiscountAmount(String userId, Double originalAmount);
```
业务描述：
1. 获取用户当前等级配置，如果没有配置则返回原价
2. 查询所有有效的折扣类型权益（状态正常、在有效期内）
3. 从会员等级折扣和权益折扣中找出最优惠的折扣率
4. 计算优惠后的金额并保留2位小数
5. 特点：
   - 支持会员等级基础折扣
   - 支持权益折扣叠加
   - 自动选择最优惠方案
   - 保证精确计算

#### 10.7.6 计算订单积分接口
```java
/**
 * 计算订单积分
 * @param userId 用户ID
 * @param amount 订单金额
 * @return 可获得的积分
 */
Integer calculateOrderPoints(String userId, Double amount);
```
业务描述：
1. 获取用户当前等级配置
2. 查询消费类型的积分规则（状态正常、在有效期内）
3. 根据规则类型计算基础积分：
   - 固定值：直接使用配置的积分值
   - 倍率：根据消费金额计算
   - 区间值：根据金额区间判断
4. 应用会员等级的积分加成
5. 检查并限制单次最高积分
6. 特点：
   - 支持多种积分计算方式
   - 考虑会员等级加成
   - 有单次积分上限控制
   - 规则必须在有效期内

#### 10.7.7 使用积分接口
```java
/**
 * 使用积分
 * @param userId 用户ID
 * @param points 积分数量
 * @param scene 业务场景
 * @param businessId 业务ID
 * @return 是否使用成功
 */
boolean usePoints(String userId, Integer points, Integer scene, String businessId);
```
业务描述：
1. 参数校验：
   - 检查积分数量是否大于0
   - 检查会员是否存在
   - 检查积分是否足够
2. 查询积分规则（用于获取积分有效期等信息）
3. 扣减积分：
   - 更新会员的积分余额
   - 更新修改时间
4. 记录积分变动：
   - 使用负数表示积分消费
   - 记录变动类型、场景、业务ID等
   - 设置积分记录的有效期
   - 添加变动说明
5. 特点：
   - 使用事务确保数据一致性
   - 积分不足时返回false
   - 详细记录变动信息
   - 支持多种业务场景

   - 应用积分倍率
4. 增加积分：
   - 更新会员的积分余额
   - 更新修改时间
5. 记录积分变动：
   - 记录变动类型、场景、规则信息等

6. 特点：

   - 详细记录积分来源和计算过程

### 10.8 业务流程设计

#### 10.8.1 会员消费流程
1. 消费前计算会员折扣金额
2. 完成支付后更新累计消费金额
3. 触发会员等级检查
4. 计算并发放消费积分
5. 记录积分变动明细

#### 10.8.2 积分使用流程
1. 检查积分余额是否足够
2. 扣减会员积分
3. 创建积分消费记录
4. 关联业务场景和规则
5. 设置积分记录有效期

#### 10.8.3 会员升级流程
1. 更新消费金额后触发检查
2. 获取所有会员等级配置
3. 根据消费金额判断适用等级
4. 如果等级发生变化则更新会员信息
5. 设置新等级有效期
6. 可选：记录等级变更历史
7. 可选：发送会员升级通知

### 10.9 注意事项

1. 金额计算
   - 数据库中金额统一使用分为单位
   - 折扣计算结果保留2位小数
   - 设置合理的金额上下限

2. 积分规则
   - 合理设置积分有效期
   - 控制单次积分上限
   - 注意并发情况下的积分计算
   - 积分变动必须有明确的来源和说明

3. 等级管理
   - 设置合理的等级晋升门槛
   - 注意等级有效期的管理
   - 考虑等级降级的场景
   - 权益的有效期和等级有效期的关系

4. 数据安全
   - 积分和金额相关操作需要加事务
   - 关键操作需要记录操作日志
   - 敏感数据需要进行加密处理
   - 接口需要进行权限控制

5. 性能优化
   - 合理使用缓存
   - 分页查询大量数据
   - 定期清理过期数据
   - 异步处理非实时任务
