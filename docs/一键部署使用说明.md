## 🖥️ 使用步骤

### 步骤1：下载脚本到服务器

```bash
# 下载位于项目根目录下的脚本文件deploy.sh
# 如果脚本文件需要做本地化修改，请先下载到本地修改后再上传
# 执行命令下载脚本文件
wget https://gitee.com/banyue0618/billiards-miniapp/blob/CueTime-Miniapp-saas-open-1.0/backend/billiards-admin/deploy.sh -O deploy.sh
```

### 步骤2：执行一键部署

```bash
# 给脚本执行权限
chmod +x deploy.sh

# 运行一键部署
./deploy.sh
```

### 步骤3：按提示配置

脚本会依次询问以下配置：

1. **🌐 域名设置**
   ```
   请输入域名（如：billiards.example.com，或直接回车使用localhost）:
   ```

2. **🚀 部署模式选择**
   ```
   部署模式选择：
   1) 简单模式（推荐新手）- 应用直接暴露8080端口
   2) 生产模式（推荐生产）- 包含Nginx反向代理+SSL
   请选择 [1-2]:
   ```

3. **📦 系统更新选项**
   ```
   系统组件更新选项：
   1) 跳过系统更新（推荐，节省时间）
   2) 更新系统包（较慢但更安全）
   请选择 [1-2]:
   ```

4. **🔒 SSL配置（仅生产模式）**
   ```
   是否配置SSL证书？[y/N]:
   ```

5. **📁 Git仓库配置**
   ```
   默认Git仓库: https://gitee.com/banyue0618/billiards-miniapp.git -b CueTime-Miniapp-saas-open-1.0
   使用默认Git仓库？[Y/n] (直接回车使用默认):
   ```

## 🎯 部署完成后

### 访问信息

**简单模式：**
- 网站地址：`http://your-domain:8080`
- 管理后台：`http://your-domain:8080/admin`
- 健康检查：`http://your-domain:8080/actuator/health`

**生产模式：**
- 网站地址：`http://your-domain`（或 `https://your-domain` 如果配置了SSL）
- 管理后台：`http://your-domain/admin`
- 健康检查：`http://your-domain/health`

### 服务管理
```bash
# 进入项目目录
cd /opt/billiards-saas/backend/billiards-admin

# 查看服务状态（根据部署模式选择配置文件）
docker-compose -f docker-compose.simple.yml ps    # 简单模式
docker-compose -f docker-compose.prod.yml ps      # 生产模式

# 查看应用日志
docker-compose -f docker-compose.simple.yml logs -f billiards-admin  # 简单模式
docker-compose -f docker-compose.prod.yml logs -f billiards-admin     # 生产模式

# 重启所有服务
docker-compose -f docker-compose.simple.yml restart  # 简单模式
docker-compose -f docker-compose.prod.yml restart    # 生产模式

# 停止所有服务
docker-compose -f docker-compose.simple.yml down    # 简单模式
docker-compose -f docker-compose.prod.yml down      # 生产模式

# 重新部署
docker-compose -f docker-compose.simple.yml up -d --build  # 简单模式
docker-compose -f docker-compose.prod.yml up -d --build    # 生产模式
```

## 🔍 故障排除

### 常见问题

**1. 端口被占用**
```bash
# 检查端口占用
netstat -tlnp | grep :8080

# 停止占用端口的服务
sudo systemctl stop apache2  # 或其他服务
```

**2. Docker权限问题**
```bash
# 将用户添加到docker组
sudo usermod -aG docker $USER

# 重新登录或执行
newgrp docker
```

**3. 服务启动失败**
```bash
# 查看详细日志
docker-compose -f docker-compose.quick.yml logs

# 检查系统资源
free -h
df -h
```

**4. 网络连接问题**
```bash
# 测试容器间连通性
docker-compose -f docker-compose.quick.yml exec billiards-admin ping mysql
docker-compose -f docker-compose.quick.yml exec billiards-admin ping redis
```

### 重新部署
如果部署失败，可以清理后重新部署：

```bash
# 停止并清理所有容器
docker-compose -f docker-compose.quick.yml down -v

# 清理Docker资源
docker system prune -f

# 重新运行部署脚本
./quick-deploy.sh
```
