## 1. 项目概览

### 1.1 项目组成

自助台球厅系统由三个主要部分组成：

- **后端API服务**：基于Spring Boot框架开发的RESTful API服务
- **管理后台**：基于Vue.js和Element UI开发的商家管理系统
- **微信小程序**：基于微信小程序框架开发的用户端应用

### 1.2 系统架构图

```
                        +----------------+
                        |   公网/CDN     |
                        +-------+--------+
                                |
+----------------+     +--------v--------+     +----------------+
| 微信小程序前端  +<--->+  后端API服务    +<--->+ 管理后台前端    |
|                |     |  (Spring Boot) |     | (Vue+Element) |
+----------------+     +--------+--------+     +----------------+
                                |
                       +--------v--------+
                       |    MySQL数据库  |
                       +--------+--------+
                                |
                       +--------v--------+
                       |    Redis缓存    |
                       +----------------+
```

#### 2.3.5 订单模块
处理用户开台和计费，包括：
- 订单创建和管理
- 实时计费逻辑
- 订单结算
- 订单统计分析

#### 2.3.6 资源存储模块

### 2.5 MyBatis Plus配置

#### 2.5.1 基础配置
```java
@Configuration
public class MybatisPlusConfig {
    
    /**
     * 分页插件配置
     */
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        // 添加分页插件
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
        // 添加乐观锁插件
        interceptor.addInnerInterceptor(new OptimisticLockerInnerInterceptor());
        return interceptor;
    }
    
    /**
     * 配置字段自动填充
     */
    @Bean
    public MetaObjectHandler metaObjectHandler() {
        return new MetaObjectHandler() {
            @Override
            public void insertFill(MetaObject metaObject) {
                this.strictInsertFill(metaObject, "createTime", LocalDateTime.class, LocalDateTime.now());
                this.strictInsertFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
            }

            @Override
            public void updateFill(MetaObject metaObject) {
                this.strictUpdateFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
            }
        };
    }
}
```

## 5. 开发规范

### 5.1 编码规范

#### 5.1.1 Java编码规范
- 采用Google Java Style
- 使用Lombok简化代码
- 方法注释必须包含功能描述、参数说明和返回值说明
- 异常需要合理处理和日志记录

#### 5.1.2 前端编码规范
- 使用ESLint和Prettier强制代码风格
- 组件采用PascalCase命名
- CSS采用BEM命名规范
- 接口调用统一通过services层处理

#### 5.1.3 API规范
- 遵循RESTful设计原则
- URL采用kebab-case命名法
- 请求/响应采用统一的格式封装
- API版本控制采用URL前缀方式

### 5.2 Git工作流

- 采用Git Flow工作流
- 主分支: main/master
- 开发分支: develop
- 功能分支: feature/*
- 发布分支: release/*
- 修复分支: hotfix/*

### 5.3 测试策略

- 单元测试: 要求核心业务逻辑覆盖率>80%
- 集成测试: 要求主流程覆盖>90%
- 前端组件测试: 关键组件需要编写测试用例
- 端到端测试: 核心业务流程需要E2E测试
